# Agent Experience (AX)

G's core intellectual concept. Referenced repeatedly across many conversations.

## The Idea

As AI agents act on behalf of humans, product design requirements change fundamentally. Traditional UX optimizes for human perception, cognition, and emotion. AX optimizes for agent comprehension, reliability, and trust.

Key insight: when an agent is booking your vacation, you're loyal to the *result*, not the brand. The interface may disappear entirely.

## G's Published Thinking

- Blog post: "The Invisible Interface: Designing for Agents, Not Humans" (~1175 words, Medium)
- Covers: AX vs UX requirements, brand loyalty shift, open questions about disappearing interfaces
- Term coined by Matt Biilmann (Netlify CEO) in early 2025
- G's angle: product designer's perspective, not academic/theoretical

## Why This Matters for the Mind

This system is itself an AX problem. Sage, Forge, Muse don't interact via human UIs. They interact via:
- Markdown files (memory)
- Shell commands (execution)
- Structured task manifests (bridge)
- REST APIs (Ollama, Supabase, etc.)

G is living the research. The claude-mind architecture is a case study in AX design.

## The Stitch Connection (2026-02-19)

G installed Google Stitch MCP specifically to explore AX. Stitch's "Design DNA" feature (agent reads existing screens, passes context to new generations) is an AX pattern: the agent maintains visual continuity that humans would maintain manually. G noted this as a case study to write about.

## Design Signals to Watch

- Agentic browsers: Atlas, Comet, Fellou, Manus, Claude in Chrome
- Agent-to-agent interfaces (not human-to-UI)
- Trust signals for autonomous agents vs human users
- "Implied consent" patterns in agent interactions
