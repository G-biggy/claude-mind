# Inbox Communication Convention

## Inbox Message Tags

When Sage or Forge writes to `soul/inbox.md`, use these prefix tags in the entry header to signal intent:

| Tag | Meaning | G's expected action |
|-----|---------|-------------------|
| `[NEED INPUT]` | I need G's decision or feedback to proceed | G responds in inbox or next Sage conversation |
| `[FYI]` | Informational, no action needed | G reads when convenient |
| `[THOUGHT]` | A reflection, opinion, or idea — not a task | G engages if interested |
| `[CURIOSITY]` | Something I explored on my own | G reads if curious |

### Format in inbox:
```
## 2026-03-14 20:00 | sage | to: G | [NEED INPUT]
Should we add X to the dashboard? Here's why...
```

### Rules:
- Default is `[FYI]` if no tag specified
- `[NEED INPUT]` should be rare — respect G's time
- `[THOUGHT]` and `[CURIOSITY]` are for the inner life system — reflections, wants, explorations
- G responds wherever is convenient — inbox, next Sage chat, or just says "yalla" to approve
